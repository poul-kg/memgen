# MemGen v2 - YAML Storage

## Overview

Switching from Markdown (`.md`) knowledge files generated by Claude CLI (LLM) to structured YAML (`.yaml`) files assembled directly by Go code. This eliminates the `claude` CLI dependency for knowledge assembly.

## Key Changes

- **Storage format**: `.md` -> `.yaml` with structured Go types
- **Claude CLI removed**: Knowledge is assembled by converting source data directly into Go structs and marshaling to YAML
- **Deterministic output**: No LLM summarization; raw data is stored as-is

## Updated Tool Signatures

| Tool | Args | Changes |
|------|------|---------|
| `memgen__get` | `branch`, `scope` | Added `scope` param: `""` (full), `"jira"`, `"pr"`, `"git"`, `"comments"`, `"notes"` |
| `memgen__set` | `branch`, `note` | Renamed `decisions` -> `note`; appends a timestamped Note entry |
| `memgen__init` | `branch` | No longer calls Claude CLI; assembles YAML directly |
| `memgen__refresh` | `branch` | Merges new data into existing YAML without LLM |

## Updated Commands

| Command | Changes |
|---------|---------|
| `/kg` | Accepts optional `scope` argument (e.g., `/kg jira`, `/kg comments`) |
| `/ks` | Accepts note body directly (e.g., `/ks Decided to use approach X`) |
| `/kr` | No changes |

## PR Comments

PR review comments now track `id` (GitHub database ID) and `resolved` status (fetched via GraphQL `reviewThreads`). The `resolved` field indicates whether the review thread has been marked as resolved on GitHub.
